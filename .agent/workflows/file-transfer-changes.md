# ファイル転送変更前チェックリスト

// turbo-all

## 1. 変更前の準備

```bash
# 安定版のタグを作成
git tag stable-backup-$(date +%Y%m%d)
git push origin stable-backup-$(date +%Y%m%d)
```

## 2. ローカルでのテスト（必須）

変更をプッシュする前に、必ずローカルで動作確認を行う：

```bash
npm run dev
```

### テスト手順：
1. ブラウザを2つ開く（または2つのタブ）
2. 両方で `http://localhost:3000` にアクセス
3. 送信側：小さなテストファイル（1KB程度のテキストファイル）を選択
4. 受信側：送信側のIDを入力して接続
5. ファイルをダウンロード
6. **ダウンロードしたファイルが開けることを確認**

## 3. 変更をプッシュする前

上記テストが成功した場合のみ：

```bash
git add .
git commit -m "説明的なコミットメッセージ"
git push
```

## 4. 本番でのテスト

Vercelデプロイ完了後（1-2分待つ）：
1. 本番URLで同じテストを実行
2. 問題があれば即座にリバート：

```bash
git revert HEAD
git push
```

## 5. 問題が発生した場合の緊急リバート

```bash
# 最新の安定版タグに戻す
git reset --hard <安定版タグ名>
git push --force
```

## 禁止事項

- ❌ ローカルテストなしでプッシュしない
- ❌ ファイル転送のコア部分を一度に大幅変更しない
- ❌ 複数の機能変更を1つのコミットにまとめない
- ❌ 暗号化/ワーカー/並列処理を同時に変更しない
